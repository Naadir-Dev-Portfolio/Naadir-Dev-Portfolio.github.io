# Skills Breakdown

This file documents how the portfolio calculates the About section skill bars.

## Source data

The browser reads `window.__PORTFOLIO.DATA` from `assets/js/data.js`. That file is generated by `scripts/compile_projects.py` from the project `portfolio/*.json` files across the GitHub org.

Each visible project can contribute to a skill by:

- its top-level `section` key, such as `python`, `powerbi`, `web`, or `mobile`
- its `section/category` pair, such as `python/automation`
- its `tags` values, normalised to lowercase with spaces changed to hyphens

A project is counted only once per skill even if it matches that skill through both a section and a tag.

## Percentage rule

Skill percentages are project-count based:

| Matching projects | Percentage |
|---:|---:|
| 1 to 7 | 10% per project |
| 8 to 11 | 5% per project after 70% |
| 12+ | 2.5% per project after 90%, rounded down |
| Any count above cap | 100% maximum |

Examples:

- 7 Python projects = 70%
- 8 Python projects = 75%
- 9 Python projects = 80%
- 10 Python projects = 85%
- 11 Python projects = 90%
- 12 Python projects = 92%
- 15+ matching projects = 100%

## Current skill groups

| Skill label | Matching source |
|---|---|
| Python | `python` section plus Python/PyQt/Tkinter/Pandas/openpyxl tags |
| VBA / Excel | `excelvba` section plus VBA, VBS, and Excel tags |
| Power BI | `powerbi` section plus Power BI tags |
| AI / Agents | `ai` section plus AI, agent, Gemini, OpenAI, LLM, ComfyUI, and Stable Diffusion tags |
| Websites | `web` section plus explicit website tags |
| Mobile Apps | `mobile` section plus React Native, Expo, mobile, Android, and Kotlin tags |
| Automation | `python/automation`, `excelvba/vba-macros`, plus automation, Power Automate, SAP, ExtendScript, and JSX tags |
| Data Visualisation | `powerbi` section plus dashboard, data-viz, PyQtCharts, and Matplotlib tags |
| Data Transformation | `excelvba/powerquery` plus Power Query, ETL, dataflow, Pandas, and openpyxl tags |
| Browser Extensions | `browserextensions` section plus browser, extension, Chrome, and Chromium tags |

The implementation lives in `assets/js/main.js` inside `initSkills()`.
