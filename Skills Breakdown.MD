# Skills Breakdown

This file documents how the portfolio calculates the About section skill bars.

## Source data

The browser reads `window.__PORTFOLIO.DATA` from `assets/js/data.js`. That file is generated by `scripts/compile_projects.py` from the project `portfolio/*.json` files across the GitHub org.

Each visible project contributes to a skill by its portfolio bucket only:

- its `section/category` pair, such as `python/automation`

Tags are deliberately ignored. This keeps each percentage tied to the number of projects displayed in the matching project tabs.

## Percentage rule

Skill percentages are project-count based:

| Matching projects | Percentage |
|---:|---:|
| 1 to 7 | 10% per project |
| 8 to 11 | 5% per project after 70% |
| 12+ | 2.5% per project after 90%, rounded down |
| Any count above cap | 100% maximum |

Examples:

- 7 Python projects = 70%
- 8 Python projects = 75%
- 9 Python projects = 80%
- 10 Python projects = 85%
- 11 Python projects = 90%
- 12 Python projects = 92%
- 15+ matching projects = 100%
